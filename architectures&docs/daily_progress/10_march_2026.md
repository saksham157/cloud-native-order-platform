## Progress Update — 10 March 2026

### Work Completed Today

#### 1. NAT Routing for Private Subnets

Extended the VPC module to allow private subnets to access the internet securely.

**Files Added**

```
nat-gateway.tf
private-route-table.tf
```

**Resources Implemented**

* Elastic IP allocation
* NAT gateway configuration
* Private route table
* Route table association for private subnets

**Why**

Resources inside private subnets (such as Kubernetes worker nodes or application servers) must be able to access the internet for tasks like downloading packages, pulling container images, and performing updates.

However, these resources should not be directly accessible from the internet.
The NAT configuration allows **outbound internet access while keeping private resources isolated from inbound traffic**.

---

#### 2. Environment-Based Deployment Structure

Created Terraform environment directories to support separate infrastructure deployments.

**Folders Created**

```
terraform/environments
   ├ dev
   ├ staging
   ├ test
   └ prod
```

**Why**

Each environment will use the same Terraform modules but provide different configuration values through `terraform.tfvars`.

This enables safe infrastructure promotion across environments such as:

```
dev → staging → production
```

and supports CI/CD pipelines for automated deployments.

---

#### 3. Terraform Variable Configuration Using tfvars

Implemented environment-specific configuration using Terraform variable files.

**Files Created**

```
variables.tf
terraform.tfvars
main.tf
```

inside the `dev` environment.

**Why**

This separates infrastructure logic from configuration values and allows each environment to define its own network configuration such as:

* CIDR ranges
* subnet allocation
* availability zones

---

#### 4. Module Integration with Environment

Connected the `dev` environment to the reusable VPC module.

**Purpose**

This allows Terraform to deploy the VPC infrastructure using module outputs and environment variables, enabling consistent infrastructure deployment across multiple environments.
