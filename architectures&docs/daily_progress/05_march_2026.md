### Architecture Decisions

The following technical decisions were taken during the initial design phase:

* The infrastructure will be deployed on **AWS Cloud**
* Kubernetes cluster will be configured using **kubeadm** instead of managed Kubernetes to gain deeper operational understanding
* Infrastructure provisioning will be implemented using **Terraform**
* Application services will be containerized using **Docker**
* CI/CD pipeline will be implemented using **Jenkins**
* Event-driven communication will use **SQS** and **SNS**
* Background processing tasks will use **AWS Lambda**
* Persistent storage will use **RDS**
* Centralized logging and monitoring will be implemented using **ELK Stack**

### System Goals

The main objectives of this project are:

* Demonstrate real-world cloud infrastructure design
* Implement infrastructure automation using Infrastructure as Code
* Deploy containerized microservices using Kubernetes
* Implement an event-driven architecture
* Design a highly available multi-AZ infrastructure
* Implement centralized logging and monitoring
* Automate deployment using CI/CD pipelines

### Problems Faced

Initial challenge in organizing a clean repository structure that separates:

* Infrastructure code
* Kubernetes configuration
* DevOps automation scripts
* Application services
* Documentation

### Solution

A modular repository structure was implemented where each major system component is organized into its own directory.

This structure improves maintainability and mirrors how large-scale DevOps projects are organized in production environments.

### Future Improvements

The following enhancements are planned for future iterations of the project:

* Implement chaos engineering tests to simulate node failures
* Add monitoring using Prometheus and Grafana
* Implement autoscaling policies in Kubernetes
* Add alerting and observability improvements
