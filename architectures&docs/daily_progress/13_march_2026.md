## Progress Update — 13 March 2026

### Work Completed Today

#### 1. IAM Module Implementation

Created a reusable Terraform module to manage AWS IAM roles required for Kubernetes infrastructure.

**Folder Created**

```
terraform/modules/iam
```

**Files Added**

```
main.tf
variables.tf
outputs.tf
```

**Resources Implemented**

* IAM role for the EKS control plane
* IAM role for EKS worker nodes
* Required AWS managed policy attachments

**Why**

EKS requires dedicated IAM roles to allow the control plane and worker nodes to interact with AWS services such as networking, container registry, and cluster management APIs.

The IAM module centralizes these permissions and allows reuse across different environments.

#### 2. EKS Cluster Module Creation

Implemented the Terraform module responsible for provisioning the Kubernetes control plane.

**Folder Created**

```
terraform/modules/eks
```

**Files Added**

```
main.tf
variables.tf
outputs.tf
```

**Resources Implemented**

* EKS cluster resource
* Integration with VPC private subnets
* Integration with cluster security group
* Integration with IAM cluster role

**Why**

The EKS control plane manages the Kubernetes cluster, scheduling workloads and coordinating communication between worker nodes and the Kubernetes API server.

Deploying the cluster inside private subnets improves security by preventing direct internet exposure.

---

#### 3. Worker Node Group Configuration

Added configuration to automatically create worker nodes for the Kubernetes cluster.

**Resources Implemented**

* Managed EKS node group
* Node scaling configuration
* EC2 instance type selection for worker nodes

**Why**

Worker nodes provide the compute capacity required to run containerized applications.
The node group enables Kubernetes workloads to be scheduled on EC2 instances within the cluster.
