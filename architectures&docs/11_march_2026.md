## Progress Update — 11 March 2026

### Work Completed Today

#### 1. Security Groups Module Creation

Implemented a reusable Terraform module to manage network security rules for infrastructure components.

**Folder Created**

```
terraform/modules/security-groups
```

**Files Added**

```
main.tf
variables.tf
outputs.tf
```

**Purpose**

Security groups are used to control inbound and outbound network traffic within the VPC.
This module centralizes security rules so they can be reused across multiple environments such as dev, staging, and test.

---

#### 2. EKS Control Plane Security Group

Created a dedicated security group for the Kubernetes control plane communication.

**Key Configuration**

* Allowed inbound **HTTPS (TCP 443)** traffic
* Allowed outbound traffic to external services

**Why**

The Kubernetes API server communicates securely over HTTPS.
Allowing port **443** enables tools such as `kubectl` and cluster components to communicate with the Kubernetes control plane.

---

#### 3. Worker Node Security Group

Implemented a security group for Kubernetes worker nodes.

**Key Configuration**

* Allowed node-to-node communication
* Enabled outbound traffic for internet access

**Special Rule**

```
self = true
```

**Why**

Kubernetes worker nodes must communicate with each other for internal cluster networking, service discovery, and container communication.
Using `self = true` restricts communication to instances that belong to the same security group instead of exposing access to the entire VPC.
